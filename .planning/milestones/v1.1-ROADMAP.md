# Milestone v1.1: Surfvid MVP

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 1–6
**Total Plans:** 13

## Overview

Full end-to-end native SwiftUI video skimming app for iPhone. Covers the complete user journey: browse camera roll → skim video with frame-accurate drag scrub → mark In/Out clip boundaries → review clips → export each as a passthrough MP4 to Camera Roll with share sheet. Phases 5 and 6 added post-initial-ship to tune scrub sensitivity and add pinch-to-zoom.

### Phase 5: Skim Sensitivity

**Goal:** Adjust the hardcoded scrub sensitivity constant so drag-to-skim feels precise enough to land on exact frames without requiring a UI toggle.
**Depends on:** Phase 2
**Plans:** 1 plan

Plans:
- [x] 05-01-PLAN.md — PX_PER_S 0.6 → 1.2; zero-tolerance seeks for exact inter-keyframe frames; device verified ✓

**Details:**
PX_PER_S constant tuned from 0.6 to 1.2 (full-width drag spans ~half the video duration at typical length). Seek tolerance changed to CMTime.zero for both toleranceBefore and toleranceAfter, enabling exact inter-keyframe frame positioning at the cost of slight seek latency. Timeline zone always scrubs (ignores zoom state) for consistent behaviour. Promoted from backlog item 999.1.

**Goal:** User can pinch to zoom into the video frame while skimming, pan the zoomed frame by dragging, and double-tap to reset — enabling precise inspection of framing and action before committing an In/Out point.

Plans:
- [x] 06-01-PLAN.md — Zoom state, pinch gesture, pan routing, indicator, clamping (Wave 1)

**Details:**
Requirements: TBD (UX feature beyond original v1 scope). @GestureState (pinchDelta, livePanDelta) for in-flight values with auto-reset; @State (zoom, committedPan) for committed values. ## Milestone Summary

**Key Decisions:**

- AVPlayer must stream from Photos asset URL — never load full file into memory (core PERF constraint)
- Export uses AVAssetExportPresetPassthrough — no re-encode (passthrough only)
- PlayerController kept alive across Skim→Review transition — avoids re-load stutter
- ZStack screen swap (not NavigationStack push) for Library→Skim — required for landscape lock
- objectWillChange forwarding: AppViewModel must forward playerController.objectWillChange so views observe PlayerController changes
- @GestureState for in-flight gesture values with auto-reset; @State for committed values (pinch-to-zoom pattern)
- Stay on ReviewView after export (no auto-nav to DoneView) — discovered during device testing
- await MainActor.run wraps all @Published mutations in startExport() Task — AVFoundation continuation resumes on background thread

**Issues Resolved:**

- Scrub cursor jump to beginning — lastDragX anchor bug; fixed in Phase 3 regression diagnosis
- MainActor export mutation bug — AVFoundation continuation resumes off main thread; fixed with await MainActor.run
- Dynamic Island overlap in ReviewView — landscape padding not applied to content layer; fixed
- Export All button visible with empty clip list — changed from .disabled to hidden; fixed

**Issues Deferred:**

- Volume button In/Out marking — v2; sandboxed gray-area hack via AVAudioSession KVO
- Per-clip trim scrubbers in review — v2; drag handles to fine-tune each clip's In/Out after marking
- Library improvements (sort/filter) — captured as backlog

**Technical Debt Incurred:**

- UIScreen.main.bounds used in clampPan() — deprecated API, acceptable for single-device personal tool
- DoneView auto-return (2.5s) still present in code — may feel rushed on slow exports; leave for v2 UX tuning
