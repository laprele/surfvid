<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UIWindowScene → AppDelegate | requestGeometryUpdate validates against supportedInterfaceOrientationsFor |
| AppViewModel.pickVideo → PlayerController.load | Async task; result dispatched to MainActor before screen transition |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | Rapid video taps — multiple overlapping pickVideo Tasks | accept | Each Task calls `playerController.load(asset:)` which calls `replaceCurrentItem` — the last one wins. AVPlayer handles concurrent replaceCurrentItem calls safely. In Phase 2, add a guard if rapid-tap produces visible issues. |
| T-04-02 | Denial of Service | Orientation lock race — screen transitions mid-rotation | accept | `lockOrientation` is called in `.onChange(of: appViewModel.screen)` which fires after the screen enum is set. The geometry update request and screen swap happen in the same runloop pass. If race is observed on device, fix: dispatch lockOrientation to `DispatchQueue.main.asyncAfter(deadline: .now() + 0.05)`. |
| T-04-03 | Tampering | LibraryCell tap — user taps during video loading transition | accept | `appViewModel.pickVideo` is idempotent for the same asset; for a different asset, `replaceCurrentItem` handles the swap. No crash risk. |
</threat_model>
