# Phase 1 — UI Design Contract
## App Shell & Video Browsing

> Visual and interaction contract for Phase 1. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Sources: prototype (surfvid-paper-library.jsx, surfvid-paper-skim-landscape.jsx), ARCHITECTURE.md, REQUIREMENTS.md, STACK.md.

## Design System

| Property | Value |
|----------|-------|
| Tool | iOS HIG / SwiftUI native |
| Preset | not applicable |
| Component library | SwiftUI built-ins: LazyVGrid / List / ScrollView / ZStack |
| Icon library | SF Symbols (system — zero dependency) |
| Font | SF Pro (system font — `.largeTitle`, `.title`, `.headline`, `.body`, `.caption` semantic styles + SF Pro Display for large numerals) |

No third-party UI packages. No SPM dependencies for v1.

## Themes

Phase 1 spans two distinct surfaces with opposite color schemes:

| Surface | Theme | When Active |
|---------|-------|-------------|
| Library screen | Light ("paper") | Portrait — browsing camera roll |
| Skim screen | Dark (full-bleed) | Landscape — video playback (Phase 2 builds on this foundation) |

> Phase 1 delivers the Library screen and the transition into the Skim screen. The Skim screen dark theme tokens are declared here so Phase 2 does not re-specify them.

## Spacing Scale

All values are iOS points (1pt = 1/163 inch on non-retina; rendered at 2× or 3× on device). Must be multiples of 4pt.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-label gaps, inner badge padding |
| sm | 8pt | Intra-cell spacing, metadata gap in video row |
| md | 16pt | Standard element spacing, button horizontal padding |
| lg | 24pt | Screen horizontal inset (matches prototype's 24px padding) |
| xl | 32pt | Section vertical breaks |
| 2xl | 48pt | Large section gaps (e.g. status bar clearance area) |
| 3xl | 60pt | Safe area top offset (matches prototype's paddingTop: 60) |

Exceptions (non-4pt-multiple values with hardware or HIG justification):

| Value | Usage | Justification |
|-------|-------|---------------|
| 44pt min | Touch targets — all interactive elements | iOS HIG minimum tap target requirement |
| 56pt × 72pt | Thumbnail tile | 4:3 portrait ratio — matches prototype dimensions |
| 16pt top + 16pt bottom | Video row vertical padding | Nearest multiple of 4 to prototype's 14pt |
| 152pt | HUD modal width | Matches prototype `SVVolumeHUDLandscape`, rounded to multiple of 4 |
| 34pt | Skim screen right safe-area inset | iOS system `safeAreaInsets.trailing` for home indicator on notchless iPhones — hardware constant, not a designer token. Do not hard-code; use SwiftUI `geometry.safeAreaInsets.trailing` or `.safeAreaInset(edge: .trailing)` at runtime. |

## Typography

All type is SF Pro. No custom fonts. SwiftUI semantic text styles automatically adjust for Dynamic Type; use them in code. Exact point values below are the default (non-accessibility) sizes for reference.

Declared weights: Regular (400) and Semibold (600) only.

| Role | SwiftUI Style | ~Size | Weight | Line Height | Usage |
|------|---------------|-------|--------|-------------|-------|
| Display | `.custom("SF Pro Display", ...)` | 38pt | Semibold (600) | 1.1 (tight) | Library screen title "Pick a recording" |
| Heading | `.title2` | 22pt | Semibold (600) | 1.2 | Section headings, clip export count toast |
| Body | `.body` | 17pt | Regular (400) | 1.5 | List row primary labels (video title); also used for subtitle/instructional copy with `Color(.secondaryLabel)` for visual differentiation |
| Caption | `.caption` | 13pt | Regular (400) | 1.3 | Metadata lines (date · duration), secondary status text |
| Monospace timecode | `.title2.monospacedDigit()` | 22pt | Semibold (600) | 1.0 | Skim screen playhead timecode (tabular numerals required) |

Additional typographic rules from the prototype:
- Letter spacing for the Display role: -1.4pt (tight, editorial feel)
- ALL CAPS at weight 600, letter-spacing +1.4pt: used only for HUD labels ("IN POINT", "OUT · CLIP SAVED"), source filter tab labels ("SURFVID" wordmark)
- Monospace font (SF Mono / `ui-monospace`) used only for debug frame labels inside the skim video surface — not in final UI

### Accent Color — Reserved For

Accent (`oklch(0.55 0.16 30)` / approximately `#C94C10` — a warm orange-red) is applied ONLY to:

1. The In-point icon inside the Volume HUD modal (skim screen)
2. The "MARKING · IN" pill text and dot indicator (skim screen)
3. Clip range highlight bars in the mini filmstrip (skim screen, `oklch(0.65 0.14 30 / 0.45)` fill + `oklch(0.7 0.16 30)` border)
4. The pending-In marker line and diamond cap in the filmstrip
5. Export confirmation checkmark circle background (Phase 4)
6. Active tab underline in Library source tab bar

Accent is NOT used for: general interactive elements, navigation buttons, disclosure chevrons, or body copy.

### Destructive Color

| Role | iOS Semantic | Hex Reference | Usage |
|------|-------------|---------------|-------|
| Destructive | `Color(.systemRed)` | `#FF3B30` | Clip delete actions only (Phase 3) |

Phase 1 has no destructive actions. Color declared for downstream phases.

## Interaction Contracts

### LIB-01 — Video Library Grid / List

The prototype uses a **list layout** (not a grid) for the library screen. Each row: thumbnail left, title + metadata right, disclosure chevron far right.

| Property | Value |
|----------|-------|
| Layout | SwiftUI `List` with `.listStyle(.plain)` |
| Sort order | Most-recently-added first (LIB-02 — `creationDate` descending) |
| Thumbnail size | 56pt × 72pt |
| Thumbnail loading | Asynchronous via `PHImageManager.requestImage` at `PHImageRequestOptionsDeliveryModeFastFormat` first, then full quality. Never block the scroll. |
| Thumbnail placeholder | Rounded rectangle filled with `Color(.secondarySystemFill)`, no spinner |
| Row tap | Single tap — triggers `pickVideo(asset:)` → transitions to Skim screen |
| Row press state | `.contentShape(Rectangle())` with system highlight (default List behavior) |
| Scroll | Plain scrollable `List`; no pagination; load all PHAssets up front (PHFetchResult is lazy) |
| Empty state | See Copywriting Contract below |

### Photos Permission Flow

| State | UI |
|-------|-----|
| `.notDetermined` | Full-screen permission prompt (see Copywriting Contract) |
| `.authorized` / `.limited` | Library grid rendered immediately |
| `.denied` / `.restricted` | Error state with Settings deep-link (see Copywriting Contract) |

Permission prompt is shown before the library grid. No custom permission pre-prompt UI in v1 — use iOS system dialog directly, triggered by `PHPhotoLibrary.requestAuthorization(for: .readWrite)`.

### Library → Skim Transition

| Property | Value |
|----------|-------|
| Mechanism | ZStack screen swap (NOT NavigationStack push) — matches ARCHITECTURE.md decision |
| Orientation | Library is portrait; Skim is landscape — app forces orientation change on screen swap |
| Animation | Default SwiftUI `.opacity` transition (0.2s ease-out) — no slide push |
| State | `AppViewModel.screen` enum drives the ZStack switch |

### Scroll Performance (PERF-01 context)

The video list itself has no performance risk — PHFetchResult is a lazy cursor; no full file loads happen in the list. Thumbnail requests must use `PHImageRequestOptions` with `isSynchronous = false` and `deliveryMode = .opportunistic`. Cancel in-flight requests when a cell leaves the screen (cancel by `requestID`).

## States Required

Each component must handle all applicable states:

### Library Screen

| State | Trigger | UI |
|-------|---------|-----|
| Loading | First launch, Photos not yet fetched | Skeleton rows (3 placeholder rows with gray rectangles at thumbnail and text positions) |
| Populated | PHFetchResult ready | Full list |
| Empty | Zero videos in camera roll | Empty state illustration + copy (see above) |
| Permission denied | User denied Photos access | Permission denied full-screen state + "Open Settings" button |
| Permission limited | User granted limited access | List renders with available assets; no special UI needed |

### Library Row Thumbnail

| State | UI |
|-------|-----|
| Loading | `Color(.secondarySystemFill)` rounded rect, no spinner |
| Loaded | `Image` scaled to fill |
| Failed | `Image(systemName: "film")` SF Symbol in `Color(.secondarySystemFill)` bg |

## Component Safety

No third-party registries. No package dependencies. All components are Apple-first-party.

| Framework | Purpose | Safety |
|-----------|---------|--------|
| SwiftUI | All UI screens and layouts | Apple first-party |
| AVFoundation | Video loading, seeking, export | Apple first-party |
| AVKit | AVPlayerLayer via UIViewRepresentable | Apple first-party |
| PhotosKit | Camera roll access, PHAsset fetch, save-back | Apple first-party |

Phase 1 specifically uses: SwiftUI, PhotosKit. AVFoundation/AVKit are initialized (PlayerController created) but full playback is Phase 2.

│                              │
│ ┌──────┬────────────────────┐│  ← Row 1 (no top separator)
│ │ 56×72│  Video Title        ││    Row padding: 16pt top/bottom
│ │ thumb│  Yesterday · 30:42  ││    Horizontal inset: lg (24pt)
│ └──────┴──────────── › ─────┘│

│ ┌──────┬────────────────────┐│  ← Row 2
