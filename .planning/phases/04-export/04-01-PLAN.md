Declare: enum ExportError: Error with cases sessionCreationFailed, assetUnavailable, photosAccessDenied, cancelled, unknown.

Declare: class ExportManager: ObservableObject with no @Published properties. Three optional callbacks stored as var properties: onProgress: ((UUID, Float) -> Void)?, onClipComplete: ((UUID, URL) -> Void)?, onClipFailed: ((UUID, Error) -> Void)?. A private var currentTimer: Timer? to hold a reference for invalidation on unexpected deinit.

Implement func exportClip(_ clip: AppViewModel.Clip, phAsset: PHAsset) async throws -> URL:
1. Call requestAVAsset(for: phAsset) to get AVAsset.
2. Build outputURL: FileManager.default.temporaryDirectory.appendingPathComponent(UUID().uuidString + ".mp4"). Pre-delete with try? FileManager.default.removeItem(at: outputURL) — per RESEARCH.md Pitfall 2.
3. Guard-create AVAssetExportSession(asset: avAsset, presetName: AVAssetExportPresetPassthrough) — throw ExportError.sessionCreationFailed on nil.
4. Set session.outputURL = outputURL, session.outputFileType = .mp4.
5. Set session.timeRange = CMTimeRange(start: CMTimeMakeWithSeconds(clip.start, preferredTimescale: 600), end: CMTimeMakeWithSeconds(clip.end, preferredTimescale: 600)). Guard clip.end > clip.start — throw ExportError.unknown if not.
6. Create progress timer: let timer = Timer(timeInterval: 0.1, repeats: true) { [weak self] _ in self?.onProgress?(clip.id, session.progress) }. Add to RunLoop.main with forMode: .common — per RESEARCH.md Pitfall 3. Assign to self.currentTimer.
7. Wrap exportAsynchronously in withCheckedThrowingContinuation: on .completed resume(); on .failed resume(throwing: session.error ?? ExportError.unknown); on .cancelled resume(throwing: ExportError.cancelled); default resume(throwing: ExportError.unknown). Invalidate timer inside the completion handler before branching on status.
8. Return outputURL after the continuation resolves.

Implement private func requestAVAsset(for phAsset: PHAsset) async throws -> AVAsset:
Use PHVideoRequestOptions with isNetworkAccessAllowed = true and deliveryMode = .highQualityFormat (not .automatic — export needs the original bitstream, not iCloud proxy). Wrap PHImageManager.default().requestAVAsset(forVideo:options:) in withCheckedThrowingContinuation. Resume with asset or throw (info?[PHImageErrorKey] as? Error) ?? ExportError.assetUnavailable.

Implement func saveToPhotoLibrary(fileURL: URL) async throws:
Check PHPhotoLibrary.authorizationStatus(for: .addOnly) — guard it is .authorized or .limited (no additional prompt needed; app holds .readWrite which is a superset). Call try await PHPhotoLibrary.shared().performChanges { @Sendable in PHAssetChangeRequest.creationRequestForAssetFromVideo(atFileURL: fileURL) }. The @Sendable closure annotation is required per RESEARCH.md Pattern 3 to avoid Swift 6 data-race warnings.

Add deinit: currentTimer?.invalidate() as a safety net.
