## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| clip.exportedURL → Share Sheet | The URL passed to UIActivityViewController must be a valid file URL in the app's temp directory; never constructed from user input |
| ReviewView → AppViewModel.startExport() | Button is gated on !isExporting; re-entrancy guard in startExport() prevents double-export |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Denial of Service | AppViewModel.startExport — re-entrancy | mitigate | Guard `!isExporting` at top of startExport(); Export All button also disabled while isExporting — double guard |
| T-04-06 | Tampering | ActivityViewController — activityItems | accept | activityItems receives clip.exportedURL which is only set after successful export to UUID-named temp file; no user-controlled URL path |
| T-04-07 | Information Disclosure | DoneView — clip count display | accept | Clip count is an integer count of user's own clips; no sensitive information disclosed |
| T-04-08 | Denial of Service | DoneView auto-return — Task.sleep | accept | Task.sleep is non-blocking; app remains responsive during 2.5s wait; user cannot get stuck (screen will always transition) |
</threat_model>
