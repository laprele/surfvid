<objective>
Add pinch-to-zoom, drag-to-pan, double-tap reset, and zoom level indicator to SkimView.
All changes are confined to SurfvidApp/Skim/SkimView.swift — no other files are modified.

Design rationale:
- Two @GestureState properties (pinchDelta, livePanDelta) handle in-flight gesture values and auto-reset on gesture end; two @State properties (zoom, committedPan) accumulate committed values.
- The single DragGesture handles both scrub (zoom == 1) and pan (zoom > 1) via routing in .updating and .onChanged — no second DragGesture needed, no gesture conflict.
- MagnificationGesture runs simultaneously with the DragGesture via .simultaneousGesture; iOS correctly routes two-finger pinch to MagnificationGesture and one-finger drag to DragGesture.
- TapGesture(count: 2).exclusively(before: TapGesture(count: 1)) ensures double-tap fires reset and single-tap fires chrome toggle with no cross-fire.
- Pan clamping uses UIScreen.main.bounds (acceptable deprecation for this personal tool on locked-orientation iPhone).
- scaleEffect + offset are applied to Layer 2 (PlayerView) only; the gesture surface (Layer 3) and chrome layers are unaffected and always occupy the full screen.
</objective>

private func clampPan() {

- `.updating($livePanDelta)` sets the in-flight pan offset when zoom > 1 (auto-resets on gesture end via @GestureState).
- `.onChanged` runs scrub logic ONLY when zoom <= 1 (guarded by `guard zoom <= 1`); pan displacement is handled entirely via .updating above.
